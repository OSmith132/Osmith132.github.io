FeedTrac

FeedTrac is a web-based feedback tracking application developed for academic institutions. It allowed students to submit detailed feedback reports, including urgency, tags, and descriptions, which could then be accessed by relevant staff members for review in an orderly and efficient manner.


The project was developed as a part of a team using HTML, CSS, and JavaScript for the frontend, while the backend was built in PHP with MariaDB managing the database. Development took place locally on an Apache server using the XAMPP software package.


I Primarily served as a backend developer, but remained adaptable as to support frontend development and avoid workflow delays. Key responsibilities included designing the database schema, optimizing queries, and creating multiple MVC classes to manage database interactions. Additionally, I developed comprehensive unit tests for each class to ensure functionality, quality, and maintainability of the codebase.


Key skills required for this project included full-stack web development, relational database creation/management, version control with GitHub, and team collaboration using AGILE methodologies to manage workflow and meet deadlines.

Graph Theory Interactive Sandbox

The Graph Theory Interactive Sandbox (GTIS) is a software application developed in Java using LibGDX's Scene 2D for an graphical interface and Gradle for efficient build automation.


This application was created to produce interactive visualizations of complex network algorithms such as Dijkstra's Algorithm and Critical Path Analysis (CPA). It serves as an educational tool designed enhancing student's understanding of algorithmic processes by enabling them to execute various algorithms on custom-built graphs.


Developing this application required a strong foundation in network theory and a deep understanding of commonly used algorithms, such as shortest path algorithms, spanning trees, and critical path methods. Additionally, knowledge of graph structures, optimization techniques, and their practical applications was essential for implementing efficient and accurate visualizations.


I developed this project independently to address the distinct lack of resources available for students to understand and practice the graph optimisation algorithms, as outlined in the A-Level Further Maths specification.

Snake Game AI (Scored 95%)

Developed in NetLogo for my second-year AI module. This project is an advanced Snake game demonstrating multiple dynamic pathfinding algorithms. The game includes unique challenges such as simultaneous snake players, walls, and various customization features.


Key features include customizable settings for player count, food quantity, and win conditions, along with 9 selectable control methods or pathfinding algorithms used by the snakes to navagate the environment. These options allow for dynamic experimentation with different AI-driven behaviors.


The player also has the option to load in their own custom maps from local CSV files. These files can be created and exported via the map editor to the right side of the game window.


The algorithms in this game incluede those such as A*, Uniform-Cost, and greedy searches, as well as the options for human controlled snaked through the use of either the keyboard or mouse location. All of these algorithms can be ran simultaneously using multiple snakes and with the option to visually display the algorithm steps in real time. I chose to include a slider that allows the user to manually adjust the weighting of greedyness when calculating the path of the A* algorithm and allow for experimentation


This project required a strong understanding of AI algorithms, particularly A*, greedy searchM, and random walk techniques, as well as effective multi-agent simulation.
